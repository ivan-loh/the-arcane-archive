Elements

The Spell Brigade declares 21 values in the ElementType enum: None, five “core” elements, five “double” intensifications, and ten dual combos. Every outgoing DamageInfo carries one packed ElementType that drives both the damage-side multipliers (via ElementEffectivenessCalculator) and the per-element side-effect classes (Burn DOT, Chill, Ricochet, Shred, Corruption).

The five base elements:

Fire Lightning Ice Acid Dark

The five core elements

Every infusion you commit is either a fresh element (turning None into a single), a double-up of the spell’s existing element (intensifies the side-effect), or a different element on top of an existing one (a dual combo where both effects can fire).

Fire — burning DOT

Single Fire (ElementType.Fire) attaches a FireEffect to the damage event. FireEffect reads its FireEffectConfiguration for three numbers:

Damage
Damage of each individual burn tick.
NumberOfBurns
How many ticks the burn applies.
SecondsBetweenBurns
Cadence between ticks.
CanCriticalHit
If true, each burn tick can independently roll for crit.

FireDouble (Fire-on-Fire infusion) intensifies the burn: higher tick damage and/or more ticks, depending on the spell’s configuration. Fire’s strength is sustained AoE pressure; its weakness is that the initial hit doesn’t carry the bonus — only the DOT tail does.

Lightning — chain ricochet

Single Lightning attaches a LightningEffect that bounces extra hits to nearby enemies via LightningEffectConfiguration:

NumberOfLightningRicochets
How many additional targets the chain finds.
LightningDamageFactor
Multiplier (0.0 – 1.0+) applied to the original damage at each bounce.

LightningDouble increases ricochets and/or factor. Lightning is the dense-pack clearer: extra effective coverage at no extra cast cost.

Acid — resistance shred

Acid is the only single-element effect that doesn’t deal extra damage — it makes the target take more damage from everything for a duration. AcidEffect applies via AcidEffectConfiguration:

AcidDuration
How long the resistance debuff lingers.
ResistanceReduction
Subtracted from the target’s damage resistance (effectively an amp on incoming damage).

AcidDouble extends duration or deepens the shred. Acid is a multiplier on your other damage — best paired with a high-DPS primary spell rather than serving as one itself.

Ice — chill / slow

Single Ice applies IceEffect, which slows the target via IceEffectHandler. Two configuration fields:

ChillDuration
How long the slow persists per hit.
ChillDamageFactor
How fast chill accumulates from incoming damage.
Chill is clamped 0.2 – 0.72

The handler uses two constants — MinChillFactor = 0.2 and MaxChillFactor = 0.72 — so the slow effect saturates between 20% and 72% reduction. At hard freeze the target is fully stopped for the duration. Stacking more Ice past the cap only extends time-in-effect, not slow magnitude.

IceDouble intensifies the chill ramp / duration. Ice is the survival pick: kited enemies hit for far less.

Dark — corruption

Dark stands apart. DarkEffect adds corruption points scaling with CorruptionDamageFactor × damage. Once the target’s corruption fills, it’s turned — the enemy enters CorruptionState.AttackOtherEnemies and fights for you until KillAfterCorruptionExpired removes it.

CorruptionDamageFactor
Multiplier converting incoming Dark damage into corruption points.
CorruptionElementBehaviour
What element the corrupted enemy’s attacks use — see below.

Damage dealt by a corrupted enemy comes back through CorruptedPlayerSpellDamageInfo, attributed to the corrupting player so kill credit, XP, and on-kill triggers still fire.

Single, Double, and Dual combos

ElementType packs the spell’s elemental identity into a single byte. There are three kinds of packed value:

Single (5 values)
One element infused into a previously-neutral spell. Fire, Lightning, Acid, Ice, Dark.
Double (5 values)
The same element infused a second time. Intensifies the per-element side-effect: stronger DOT, more ricochets, deeper shred, longer chill, more corruption. FireDouble, LightningDouble, AcidDouble, IceDouble, DarkDouble.
Dual combo (10 values)
Two different elements layered. Both element-effect classes fire — a FireLightning spell deals burn ticks and chains ricochet damage. The combo identifier orders the two elements by enum id (Acid < Dark < Fire < Ice < Lightning), so an Acid-then-Fire and a Fire-then-Acid both pack as AcidFire.
Detecting components

ElementTypeExtensions.IsInfusedWith(ElementType, ElementType) answers “does this packed element contain X?” That’s how the game decides whether to fire the matching IElementEffect when a hit lands. ContainsDark is the convenience wrapper used by the corruption pipeline.

Effectiveness math

On the damage number side, elements only do two things via ElementEffectivenessCalculator: 

// Skip everything if no element
if damageInfo.Element == None: return value

// 1. Attacker-side bonus  (StatType 30, PercentAdd)
value ×= (1 + attacker.ElementStrengthMultiplier / 100)

// 2. Target-side bonus    (StatType 27, PercentAdd; only if target has it)
value ×= (1 + target.ElementWeaknessMultiplier   / 100)

That’s the entire elemental damage scaling. There is no built-in “Fire deals +X% to Ice enemies” matrix — effectiveness comes from per-player stats. Enemies typically don’t carry ElementWeaknessMultiplier at all, so step 2 commonly degenerates to ×1.0 against trash mobs. Players (under specific curses / traits) and certain bosses are the usual carriers.

Worked example

An attacker with ElementStrengthMultiplier = 80 firing a Fire spell against an enemy with ElementWeaknessMultiplier = 0 applies ×(1 + 0.8) × (1 + 0) = ×1.8. The same attacker against a player teammate carrying ElementWeaknessMultiplier = 25 from a curse applies ×1.8 × 1.25 = ×2.25. (Friendly-fire ×0.25 still happens later on the player side.)

Corruption behaviour

When Dark fills a target’s corruption meter, the resulting enemy attacks other enemies. The element it uses is controlled by CorruptionElementBehaviour:

UseCoreElement
The corrupted enemy uses the originating spell’s core element.
UseNoElement
The corrupted enemy deals non-elemental damage.
UseDarkElement
The corrupted enemy always deals Dark damage (spreads corruption).
