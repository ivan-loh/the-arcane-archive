Armor & Dodge

The defensive layer for a player. After all attacker-side scaling (base damage, crit, element strength, debuffs), the hit lands on PlayerHealthContainer.RemoveHealth, which applies — in order — dodge (binary), armor (flat subtract), friendly-fire scaling, and the corruption multiplier. Then HP is removed.

Armor

Armor (stat 14, Flat) is applied by PlayerHealthContainer.CalculateDamageAfterArmor. The dump only exposes the method signature, but the most likely formula — and the one consistent with the rest of the system’s design — is a flat subtraction: 

CalculateDamageAfterArmor(incoming) {
    return max(0, incoming − Armor)
}
Confidence: inferred

The alternative is the percentage model incoming × base / (base + Armor) (used in some ARPGs). Three things point at flat subtraction instead: (a) Armor is a Flat-calc stat, not a percent; (b) the artifact IncreaseDamagePerArmor only makes economic sense if armor doesn’t have built-in diminishing returns to fight against; (c) the Vampire-Survivors-derived genre defaults to flat subtraction.

Why a flat-subtract model matters

A flat-subtract model has two non-obvious properties:

  1. It can clamp to zero. If an enemy’s smallest tick is < Armor, that enemy effectively can’t hurt you. Pile enough armor and trash mobs become harmless.
  2. It scales linearly — never diminishes. Every additional armor point is exactly as good as the last (provided incoming damage is still above zero). That’s why IncreaseDamagePerArmor exists: high armor stacks without built-in falloff, so the design needs an explicit conversion option.

Dodge

Dodge (stat 15, Flat, displayed as %) is a binary roll before the armor step. If it succeeds, the hit is ignored entirely — zero HP loss, no popup-side number, and the PlayerDodgePopup shows instead. 

PlayerHealthContainer.HasDodged() {
    return GetRandomFloat(0, 100) < Dodge
}

// Inside RemoveHealth:
if (HasDodged()) {
    fire DodgeAttackEventArgs
    CheckHealOnDodgeArtifactEffect()
    return 0      // hit fully nullified
}
No hard cap visible

Nothing in StatCalculator or the dodge method clamps the value. A stat at or above 100 rolls a guaranteed dodge on every incoming hit. Content design (item availability, curses) keeps players away from that ceiling in practice.

HealOnDodge artifact hook

PlayerHealthContainer.CheckHealOnDodgeArtifactEffect reads the HealOnDodge stat (id 22, Flat) after every successful dodge and heals that many HP. The stat aggregates with any modifier that targets it. Result: dodge-stacking builds heal every time they avoid a hit, turning Dodge into a soft healing engine.

Friendly fire

FriendlyFireMultiplier = 0.25

When the attacker resolves to a teammate (most often through area / explosion damage), PlayerHealthContainer applies a hard ×0.25 multiplier from the const FriendlyFireMultiplier (). This sits between the attacker-side scaling and the recipient’s own dodge / armor — so teammates still get to dodge a 25%-strength friendly-fire hit, and armor still subtracts from it.

ExplosionDamageInfo additionally carries separate DamageOnEnemies and DamageOnPlayers fields, so an explosion can deal entirely different base numbers to each side before the ×0.25 is applied. Designers use this to make some AoEs ally-safe even at maximum stacking.

Corruption multiplier

Damage taken while the player is corrupted multiplies by the serialized field damageTakenWhileCorruptedMultiplier on PlayerHealthContainer. The exact default value isn’t dump-visible (it’s configured per-character or per-run via inspector serialization), but the multiplier sits late in the pipeline, stacking with friendly-fire if both fire on the same hit.

Worked examples

1. Plain enemy hit, light armor

incoming = 42
Dodge    = 15
Armor    = 8

roll     = RandomFloat(0,100) = 63.1   // 63.1 < 15 false → no dodge
result   = max(0, 42 − 8)        = 34   // 34 HP removed

2. Successful dodge

incoming = 42
Dodge    = 60
HealOnDodge = 3

roll     = 14.7     // 14.7 < 60 → dodge!
result   = 0        // no HP lost
heal     = +3 HP    // HealOnDodge fires

3. Trash-mob threshold (armor clamps to zero)

incoming = 6
Armor    = 10

result   = max(0, 6 − 10)      = 0   // trash mob can't break through

4. Friendly explosion through armor

incoming     = 80       // (after attacker-side scaling)
friendly     = true
Dodge        = 10
Armor        = 8

step1: ×0.25                = 20
step2: dodge roll = 87       → no dodge
step3: max(0, 20 − 8)        = 12   // final HP removed
Build implication

Armor alone is great against small-tick swarms (clamps to zero), but degrades against single big hits (a boss slam still goes through). Dodge is the inverse — useless against guaranteed-hit zone DOT, but binary salvation against a single big spike. Mixed defensive builds layer both because they fail in different conditions, not because they stack additively.
