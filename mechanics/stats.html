Stats reference

The Spell Brigade exposes exactly 36 stats via the StatType enum. Each stat is a small object: a numeric baseValue plus a List<StatModifier>. When the game asks for the current value, it calls Stat.GetValue(), which routes through StatCalculator using the stat’s declared calculation type (Flat, PercentAdd, or PercentAddLogarithmic).

In one sentence

Every +25% Damage roll you pick adds one flat modifier with StatType = Damage and Value = 0.25. At hit time, StatCalculator sums all such modifiers and applies them to the spell’s base damage as base × (1 + Σ%).

How stats work

A stat is built from a BaseStat entry on a spell or character configuration. At equip time, BaseStat.CreateCalculator() seeds a fresh Stat with a baseValue and the canonical calculation type for that StatType. From then on, modifiers are added to the list, not summed up front. Aggregation happens lazily every time something reads the stat. 

// Pseudocode of Stat.GetValue()
GetValue() = StatCalculator.Calculate(
              baseValue,
              Modifiers,
              statCalculationType
            )

The StatModifier base class is abstract; concrete subclasses include flat modifier (a fixed Value set once), StartingStatModifier (the same but tagged as baseline kit), HiddenStatModifier (numeric but never shown in UI), and ~22 computed modifier subclasses whose GetValue() is recomputed on the fly. See the stat modifiers page for the full tree.

Calculation types

Each stat is hard-bound to one of three calculation modes by StatTypeExtensions.GetCalculationType. You cannot, for example, re-route Damage through Logarithmic — the binding is static.

The three StatCalculationType modes
ID Name Formula Confidence
0 PercentAdd value = baseValue × (1 + Σ modifier.GetValue()) high
1 Flat value = baseValue + Σ modifier.GetValue() high
2 PercentAddLogarithmic value = baseValue × (1 + Σ modifier.GetValue())0.5714286 inferred
Why 0.5714286?

StatCalculator contains exactly one float constant — Exponent = 0.5714286 (≈ 4/7). This is the classic Risk-of-Rain-style concave-falloff curve and only appears on Range and Size. It means stacking +600% Size still only gets you about 70.5714 ≈ 3.21× your starting radius, not 7×. Diminishing returns by design.

Reserved IDs

Missing IDs: 21, 28, 29, 31, 32

These five enum slots are reserved but no StatType entry is declared at them. They’re almost certainly deprecated or cut stats; keeping the numeric slots open lets old save data continue to parse cleanly. Treat them as unused — they do not appear in any StatCalculator call or modifier in the dump.

Display types

How each stat is rendered in tooltips and store cards is decided by StatDisplayType:

ModifierPercent
Shown as +25% or −15%. Used for almost every PercentAdd / Logarithmic stat.
Value
Plain integer or float (e.g., Armor: 4, Rerolls: 2). Used for Flat stats whose unit is a count.
ValuePercent
A number that is itself a percentage — CriticalChance: 35%. Used by CriticalChance, CriticalDamage, Dodge, HealthPercentageOnRevive.

Core stat blurbs

Twelve stats carry most of the build weight. Quick notes:

Damage PercentAdd
Per-spell, not character-wide; every spell snapshots its own Damage stat. Equipping an item with +25% Damage appends one flat modifier with Value = 0.25; the spell’s outgoing damage becomes base × (1 + Σ%).
FireRate PercentAdd
Direct multiplier on shots per second. Stacks additively, so two +50% FireRate picks give exactly 2× cadence (not 2.25×).
Speed PercentAdd
Projectile travel speed, not player movement. Important for hitting fast-moving wave enemies.
CriticalChance Flat · 0–100
Snapshotted into the PlayerSpellDamageInfo at cast time. The roll itself happens on hit (Hitbox.IsCriticalDamage) via GetRandomFloat(0,100) < CriticalChance.
CriticalDamage Flat · %
On a successful crit, damage multiplies by (1 + CriticalDamage / 100). With CriticalDamage = 150, a crit hits for 2.5×. Crit Mastery ascension converts excess CriticalChance above 100 into bonus CriticalDamage.
Armor Flat
Flat subtraction: damage_after = max(0, incoming − Armor). The artifact IncreaseDamagePerArmor only makes sense because armor scales linearly without diminishing returns — see Armor & Dodge.
Dodge Flat · %
A successful dodge ignores 100% of the hit and fires the DodgeAttackEventArgs event. The HealOnDodge stat (id 22) can heal HP on every successful dodge.
MovementSpeed PercentAdd
Player walking speed; multiplied further by TerrainSpeedMultiplier (id 25) on hostile terrain. Several artifacts trade this off — DecreaseMovementSpeedPerArmor, LessHPMoreMovementSpeed.
MaxHealth PercentAdd
Multiplier on the character’s configuration MaxHealth. New cap is computed at level-up and on equip, then any current-HP delta is preserved.
Luck Flat
Plain integer fed into RandomRarityPicker. The exact weighting curve isn’t exposed, but every extra Luck point biases the per-pick rarity roll toward higher tiers — see Rarity.
XPGain PercentAdd
Multiplier on XP from kills. The artifact BonusExpWhenBelowHealthThreshold appends a dynamic XPGain modifier that only contributes while you’re under a threshold.
ElementStrengthMultiplier PercentAdd
The attacker-side bonus for non-None elemental hits. Multiplied after base damage and crit by ElementEffectivenessCalculator.ApplyAttackerElementStrength. Pairs with ElementWeaknessMultiplier (id 27) on the target.
Example — equipping a Damage roll

You buy an Epic +25% Damage item from the store. The game creates one flat modifier { StatType = Damage, Value = 0.25 } and calls IPlayerStats.AddCharacterImprovement. Next time any spell asks for its Damage stat, StatCalculator.CalculatePercentAddValue walks the modifier list, sums every +% contribution, and returns baseValue × (1 + Σ%). Nothing is precomputed: every read is fresh, which is why dynamic modifiers (revives-performed counter, party-members-nearby) can feed back into the formula every frame.
